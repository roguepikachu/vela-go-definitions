What is defkit?

defkit is a Go SDK that lets platform engineers author KubeVela X-Definitions — ComponentDefinition, TraitDefinition, PolicyDefinition, WorkflowStepDefinition — using native Go code. The Go code compiles to CUE transparently: you never need to write or see CUE.

🛡️
Type Safety
Catch configuration errors at Go compile time, not at Kubernetes deploy time. Full type inference for all parameter types.
💡
IDE Support
Full autocompletion, inline documentation, and refactoring in any Go IDE. No CUE plugin required.
🧪
Testable
Unit-test definitions with standard go test and Gomega matchers. No cluster needed for unit tests.
📦
Distributable
Share definitions as standard Go packages via go get. Version-controlled alongside your platform code.
🔄
CUE Compatible
Produces the same CUE output consumed by the KubeVela controller. Mix Go-authored and CUE-authored definitions freely.
⚡
Fluent API
Method-chaining builder pattern makes definitions readable and concise. No YAML, no CUE schema wrestling.
ℹ️ Both defkit (Go) and the native CUE approach produce identical output consumed by the KubeVela controller. Choose defkit when your team is Go-proficient or when definitions require complex conditional logic and unit testing.

When to Use Go vs CUE

Scenario Go (defkit) CUE
Team already proficient in Go ✓ Recommended Works too
Complex conditional template logic ✓ Clearer structure Possible, verbose
Unit testing without a cluster ✓ go test + Gomega No native support
Share definitions as packages ✓ go get Manual copying
Simple definitions, few parameters Works fine ✓ Simpler to start
Quick one-off experiments Slightly more setup ✓ Less boilerplate

Quick Start

From zero to a working KubeVela ComponentDefinition in 4 steps.

1
Initialize a definition module
Use the vela CLI to scaffold the Go module with the correct directory structure.
bash
vela def init-module --name my-platform \
  --components webservice,worker \
  --traits scaler
2
Write a definition in Go
Open the scaffolded file and use the defkit fluent API to define parameters and template.
Go — defkit
package components

import "github.com/oam-dev/kubevela/pkg/definition/defkit"

func Webservice() *defkit.ComponentDefinition {
    image    := defkit.String("image").Required()
    replicas := defkit.Int("replicas").Default(1)

    return defkit.NewComponent("webservice").
        Description("Web service component").
        Workload("apps/v1", "Deployment").
        Params(image, replicas).
        Template(webserviceTemplate)
}

func webserviceTemplate(tpl *defkit.Template) {
    vela := defkit.VelaCtx()
    tpl.Output(
        defkit.NewResource("apps/v1", "Deployment").
            Set("metadata.name", vela.Name()).
            Set("spec.replicas",
                defkit.Reference("replicas")),
    )
}

func init() { defkit.Register(Webservice()) }
CUE — generated
webservice: {
  type: "component"
  description: "Web service component"
  attributes: workload: definition: {
    apiVersion: "apps/v1"
    kind:       "Deployment"
  }
}
template: {
  output: {
    apiVersion: "apps/v1"
    kind: "Deployment"
    metadata: name: context.name
    spec: replicas: parameter.replicas
  }
  parameter: {
    // +usage=Container image
    image:    string
    replicas: *1 | int
  }
}
3
Validate & generate
Validate the module compiles and generates valid CUE.
bash
vela def validate-module ./my-platform
vela def list-module ./my-platform
4
Apply to cluster
Deploy all definitions to your KubeVela cluster in one command.
bash
# Dry-run first
vela def apply-module ./my-platform --dry-run

# Apply (overwrite if definitions already exist)
vela def apply-module ./my-platform --conflict overwrite

│   ├── env.go               # TraitDefinition — environment variables

Registration Pattern

definition file
package components

import "github.com/.../defkit"

func Webservice() *defkit.ComponentDefinition {
    return defkit.NewComponent("webservice").
        Description("...").
        Workload("apps/v1", "Deployment").
        Params(...)
}

// init() is called automatically on import
func init() {
    defkit.Register(Webservice())
}
cmd/register/main.go
package main

import (
    "fmt"
    "github.com/.../defkit"

    // Blank imports trigger init() in each package,
    // registering all definitions automatically
    _ "my-platform/components"
    _ "my-platform/traits"
    _ "my-platform/policies"
    _ "my-platform/workflowsteps"
)

func main() {
    // Exports all registered definitions as JSON
    fmt.Println(defkit.ToJSON())
}

Integration

defkit integrates with the KubeVela ecosystem through well-defined extension points. Each definition type maps to a Kubernetes CRD managed by the KubeVela controller.

🧩
ComponentDefinition
Defines a workload type. Maps Go template to Kubernetes resources (Deployment, StatefulSet, Job, etc.). End users reference by name in Application YAML.
core.oam.dev/v1beta1
🎛️
TraitDefinition
Patches or augments a workload. Uses CUE patch semantics to modify the primary output (e.g., inject env vars, set replica count, add sidecars).
core.oam.dev/v1beta1
📜
PolicyDefinition
Controls deployment topology and behavior. Used for multi-cluster override, garbage collection rules, and apply-once semantics.
core.oam.dev/v1alpha1
⚡
WorkflowStepDefinition
Defines automation steps for the application workflow engine. Steps run sequentially or in parallel as part of Application deployment.
core.oam.dev/v1beta1
🔍
Health Evaluation
defkit's Health DSL generates status.customStatus and status.healthPolicy CUE blocks. The controller uses these to determine component health state.
HealthPolicy CUE
🌐
vela CLI Integration
The vela def subcommands handle compilation, validation, and deployment. apply-module compiles Go → CUE → YAML and applies to cluster in one step.
vela def apply-module

Definition Builders

Top-level factory functions that create the four definition types. Each returns a builder with a fluent chain for configuring the definition.

defkit.NewComponent(name string) *ComponentDefinition
NewComponent
Creates a ComponentDefinition builder. Returns a fluent chain to configure the workload type, parameters, and CUE template function.
Type: Component

Chain methods available on *ComponentDefinition:

Method Description
.Description(text)Human-readable description shown in vela show.
.Workload(apiVersion, kind)Sets the fixed Kubernetes workload GVK. Use when the workload type is always the same (e.g., apps/v1, Deployment).
.AutodetectWorkload()Omits workload GVK — KubeVela detects it from the template output. Use for k8s-objects, ref-objects.
.Params(params...)Registers one or more parameter definitions built with the parameter builder functions.
.Template(func(tpl))Assigns the CUE template function that builds runtime Kubernetes resources.
.Helper(name, param)Registers a helper variable available inside the template function.
.CustomStatus(status)Attaches a custom status DSL expression to the definition.
.HealthPolicy(health)Attaches a health evaluation DSL expression. Use preset builders like DeploymentHealth().
.WithImports(imports...)Adds CUE package import paths required by the template body.
Go — defkit
func Webservice() *defkit.ComponentDefinition {
    image := defkit.String("image").Required().
        Description("Container image to run")
    replicas := defkit.Int("replicas").Default(1).
        Description("Number of replicas")

    return defkit.NewComponent("webservice").
        Description("HTTP microservice backed by a Deployment").
        Workload("apps/v1", "Deployment").
        Params(image, replicas).
        Template(webserviceTemplate)
}

func init() { defkit.Register(Webservice()) }
CUE — generated
// KubeVela generates this schema from the builder:
parameter: {
    image:    string
    replicas: *1 | int
}

// and the template function becomes the CUE
// output/patch body compiled by the controller
defkit.NewTrait(name string) *TraitDefinition
NewTrait
Creates a TraitDefinition builder. Traits patch or augment existing component workloads. They can target specific workload types and optionally mutate container specs.
Type: Trait

Chain methods available on *TraitDefinition:

Method Description
.Description(text)Human-readable description of the trait's purpose.
.AppliesTo(workloads...)Restricts which component types this trait can attach to (e.g., "webservice", "daemon"). Omit for universal traits.
.ConflictsWith(traits...)Declares mutual exclusion with other named traits.
.PodDisruptive(bool)Marks the trait as pod-disruptive (triggers pod restart when changed).
.Stage(stage)Sets the dispatch stage for ordering trait application.
.Labels(map)Attaches metadata labels to the TraitDefinition CRD.
.Params(params...)Registers typed parameter definitions.
.Template(func(tpl))Assigns the patch template function. For container mutation use tpl.UsePatchContainer(config) inside.
.Helper(name, param)Registers a helper variable for use in the template function.
Go — defkit
func Env() *defkit.TraitDefinition {
    envs := defkit.Array("env").Of(defkit.Struct("envItem").
        WithFields(
            defkit.String("name").Required(),
            defkit.String("value").Optional(),
        )).Description("Environment variables to inject")

    return defkit.NewTrait("env").
        Description("Inject environment variables into containers").
        AppliesTo("webservice", "daemon", "worker").
        PodDisruptive(true).
        Params(envs).
        Template(envTemplate)
}

func init() { defkit.Register(Env()) }
CUE — generated
parameter: {
    env?: [...{
        name:   string
        value?: string
    }]
}
defkit.NewPolicy(name string) *PolicyDefinition
NewPolicy
Creates a PolicyDefinition builder. Policies are applied globally to applications and have no workload coupling. They control lifecycle behaviors like garbage collection, apply-once semantics, or override rules.
Type: Policy

Chain methods available on *PolicyDefinition:

Method Description
.Description(text)Human-readable description of the policy behavior.
.Params(params...)Registers typed parameter definitions for the policy.
.Helper(name, param)Registers a named helper variable for the template body.
Go — defkit
func ApplyOnce() *defkit.PolicyDefinition {
    enable := defkit.Bool("enable").Default(true).
        Description("Disable KubeVela from re-applying components on every reconcile")
    rules := defkit.Array("rules").Of(defkit.Struct("rule").
        WithFields(
            defkit.StringList("selector").Optional(),
        )).Optional()

    return defkit.NewPolicy("apply-once").
        Description("Prevent KubeVela from re-applying resources that already exist").
        Params(enable, rules)
}

func init() { defkit.Register(ApplyOnce()) }
CUE — generated
parameter: {
    enable: *true | bool
    rules?: [...{
        selector?: [...string]
    }]
}
defkit.NewWorkflowStep(name string) *WorkflowStepDefinition
NewWorkflowStep
Creates a WorkflowStepDefinition builder. Workflow steps describe actions in application delivery pipelines — deploying components, running builds, applying Terraform configs, or checking metrics.
Type: WorkflowStep

Chain methods available on *WorkflowStepDefinition:

Method Description
.Description(text)Human-readable description of what the step does.
.Category(name)Groups the step under a category in the KubeVela UI and CLI.
.Scope(scope)Sets execution scope. Typically "Application" for delivery steps.
.Params(params...)Registers typed parameter definitions for the step.
.Template(func(tpl))Assigns the CUE template function executed when the step runs.
.WithImports(imports...)Adds CUE package import paths used by the template body.
Go — defkit
func ApplyComponent() *defkit.WorkflowStepDefinition {
    comp := defkit.String("component").Required().
        Description("Name of the component to apply")

    return defkit.NewWorkflowStep("apply-component").
        Description("Apply a single component from the Application").
        Category("Deploy").
        Scope("Application").
        Params(comp).
        Template(applyComponentTemplate)
}

func init() { defkit.Register(ApplyComponent()) }
CUE — generated
parameter: {
    component: string
}

Special Methods

.AutodetectWorkload() *ComponentDefinition
AutodetectWorkload
Omits a fixed workload GVK. Use when the component template can produce different resource kinds depending on parameters (e.g., k8s-objects accepts arbitrary manifests, ref-objects references existing resources).
Applies to: Component
Go — defkit
// Use AutodetectWorkload when the output kind varies
return defkit.NewComponent("k8s-objects").
    Description("Deploy arbitrary Kubernetes resources").
    AutodetectWorkload().   // no fixed GVK — KubeVela detects
    Params(objects).
    Template(k8sObjectsTemplate)
When to use vs .Workload()
// .Workload("apps/v1", "Deployment")
// → use when the component always produces
//   exactly one fixed resource kind

// .AutodetectWorkload()
// → use when template may output multiple
//   different resource kinds, or zero
//   (k8s-objects, ref-objects patterns)
.RawCUE(cueTemplate string) *ComponentDefinition
RawCUE
Injects a raw CUE string as the template body, bypassing the fluent resource builder entirely. Use when the CUE logic cannot be expressed with the builder DSL (e.g., highly dynamic object iteration). Mutually exclusive with .Template().
Applies to: Component
Go — defkit
rawBody := `
    outputs: {
        for i, obj in parameter.objects {
            "object-\(i)": obj
        }
    }
`

return defkit.NewComponent("k8s-objects").
    AutodetectWorkload().
    Params(objects).
    RawCUE(rawBody)   // bypasses .Template()
Raw CUE body injected verbatim
// The string is placed directly into the
// generated CUE definition file.
// You get full CUE power but lose the
// type-safe builder DSL.
//
// Use RawCUE only when the fluent builder
// cannot express what you need.
.WithImports(imports ...string) *ComponentDefinition
WithImports
Adds CUE package import paths to the generated definition file. Required when your template body references CUE standard library packages (e.g., strings, math) or custom KubeVela packages. Available on ComponentDefinition and WorkflowStepDefinition.
Applies to: Component WorkflowStep
Go — defkit
return defkit.NewComponent("my-component").
    Description("Uses CUE strings package").
    Workload("apps/v1", "Deployment").
    WithImports("strings").  // adds: import "strings"
    Params(name).
    Template(myTemplate)
CUE — generated header
import (
    "strings"
)

// template body can now use:
//   strings.ToLower(parameter.name)

Parameter Builders

Construct typed parameters that map directly to CUE types. Parameters are defined once and referenced in template functions by the captured Go variable.

Scalar Types

defkit.String(name string) *Param
String
A UTF-8 string parameter. Optional by default — use .Required() to enforce presence.
Applies to:All Definition Types
Go — defkit
image   := defkit.String("image").Required()
tag     := defkit.String("tag").Default("latest")
comment := defkit.String("comment") // optional, no default
CUE — generated
image:    string
tag:      *"latest" | string
comment?: string
defkit.Int(name string) *Param
Int
A 64-bit integer parameter. Supports .Min(n) and .Max(n) for range constraints.
Applies to:All Definition Types
Go — defkit
replicas := defkit.Int("replicas").Default(1).Min(1).Max(50)
port     := defkit.Int("port").Required()
CUE — generated
replicas: *1 | int >=1 & <=50
port:     int
defkit.Bool(name string) *Param
Bool
A boolean parameter. .Default(true) or .Default(false) sets the CUE default value.
Applies to:All Definition Types
Go — defkit
enable := defkit.Bool("enable").Default(true)
debug  := defkit.Bool("debug").Default(false)
CUE — generated
enable: *true  | bool
debug:  *false | bool

Collection Types

defkit.StringList(name string) *Param
StringList
A list of strings. Shorthand for Array("x").Of(String). Optional by default.
Applies to:All Definition Types
Go — defkit
args    := defkit.StringList("args")
secrets := defkit.StringList("secrets").Required()
CUE — generated
args?:   [...string]
secrets: [...string]
defkit.Array(name string).Of(paramType) *Param
Array
A typed list where every element matches the given parameter type. Chain .Of() with any scalar, struct, or complex type. Use when items have a defined schema.
Applies to:All Definition Types
Go — defkit
envItem := defkit.Struct("envItem").WithFields(
    defkit.String("name").Required(),
    defkit.String("value").Optional(),
)
envs := defkit.Array("env").Of(envItem)
CUE — generated
env?: [...{
    name:   string
    value?: string
}]
defkit.List(name string) *Param
List
An untyped list (CUE [...]). Use when items are heterogeneous or the schema is provided externally. Supports .WithFields() for a named-field variant of the list item schema.
Applies to:All Definition Types
Go — defkit
objects := defkit.List("objects").Required()
CUE — generated
objects: [...]
defkit.Object(name string).WithFields(params...) *Param
Object
A structured object with a known set of named fields. Use .WithFields() to specify the field parameter definitions. Prefer Struct when used as an element type inside Array.Of().
Applies to:All Definition Types
Go — defkit
resources := defkit.Object("resources").WithFields(
    defkit.Object("limits").WithFields(
        defkit.String("cpu").Default("500m"),
        defkit.String("memory").Default("256Mi"),
    ),
    defkit.Object("requests").WithFields(
        defkit.String("cpu").Default("100m"),
        defkit.String("memory").Default("128Mi"),
    ),
)
CUE — generated
resources?: {
    limits?: {
        cpu:    *"500m"  | string
        memory: *"256Mi" | string
    }
    requests?: {
        cpu:    *"100m"  | string
        memory: *"128Mi" | string
    }
}
defkit.StringKeyMap(name string) *Param
StringKeyMap
A map with arbitrary string keys and string values. Maps directly to the CUE [string]: string constraint. Useful for labels, annotations, or environment variable maps.
Applies to:All Definition Types
Go — defkit
labels      := defkit.StringKeyMap("labels")
annotations := defkit.StringKeyMap("annotations")
CUE — generated
labels?:      [string]: string
annotations?: [string]: string

Complex Types

defkit.Enum(name string).Values(vals ...string) *Param
Enum
A string parameter constrained to a fixed set of allowed values. Use .Values() — not .Enum() — to specify the allowed set (current convention as of defkit API update).
Applies to:All Definition Types
Go — defkit
// Use .Values(), not .Enum() — current convention
pullPolicy := defkit.Enum("imagePullPolicy").
    Values("Always", "Never", "IfNotPresent").
    Default("IfNotPresent")
CUE — generated
imagePullPolicy: *"IfNotPresent" | "Always" | "Never" | "IfNotPresent"
defkit.Struct(name string).WithFields(params...) *Param
Struct
An inline struct type with named fields. Identical semantics to Object but conventionally used as the element type inside Array.Of(). Can nest other parameter types including other Structs.
Applies to:All Definition Types
Go — defkit
portStruct := defkit.Struct("port").WithFields(
    defkit.Int("containerPort").Required(),
    defkit.Enum("protocol").Values("TCP", "UDP").Default("TCP"),
    defkit.String("name").Optional(),
)
ports := defkit.Array("ports").Of(portStruct)
CUE — generated
ports?: [...{
    containerPort: int
    protocol:      *"TCP" | "TCP" | "UDP"
    name?:         string
}]
defkit.OneOf(name string).Variants(params...) *Param
OneOf
A discriminated union — the parameter must match exactly one of the declared variant schemas. Each variant is defined with defkit.Variant(name). Maps to a CUE disjunction of struct types.
Applies to:All Definition Types
Go — defkit
source := defkit.OneOf("source").Variants(
    defkit.Variant("git").WithFields(
        defkit.String("url").Required(),
        defkit.String("branch").Default("main"),
    ),
    defkit.Variant("image").WithFields(
        defkit.String("ref").Required(),
    ),
)
CUE — generated
source?: {
    url:    string
    branch: *"main" | string
} | {
    ref: string
}

Chain Methods

All parameter constructors return *Param — chain these methods to configure validation, defaults, and metadata. Methods are composable in any order.

.Required() / .Optional() / .ForceOptional()
Required / Optional / ForceOptional
Controls whether the parameter must be supplied. Parameters are optional (x? in CUE) by default. .Required() makes the field mandatory. .ForceOptional() overrides Required back to optional — useful for overriding inherited schemas.
Go — defkit
defkit.String("image").Required()        // mandatory
defkit.String("tag")                     // optional (default)
defkit.String("tag").Optional()          // explicit optional
defkit.String("name").ForceOptional()    // override to optional
CUE — generated
image: string     // required — no ?
tag?:  string     // optional — has ?
name?: string     // force-optional — has ?
.Default(value interface{}) *Param
Default
Sets a default value. The CUE default marker (*) is placed before the value in the disjunction. Works with strings, ints, bools, and string slices.
Go — defkit
defkit.String("tag").Default("latest")
defkit.Int("replicas").Default(1)
defkit.Bool("readOnly").Default(false)
CUE — generated
tag:      *"latest" | string
replicas: *1        | int
readOnly: *false    | bool
.Description(text string) / .Short(letter rune) *Param
Description / Short
.Description() adds a human-readable description shown in vela show and generated OpenAPI schemas. .Short() assigns a single-letter CLI shorthand flag (e.g., -i for image).
Go — defkit
defkit.String("image").
    Required().
    Short('i').
    Description("Container image reference (e.g. nginx:1.25)")
CUE — generated annotation
// +usage=Container image reference (e.g. nginx:1.25)
// +short=i
image: string
.Values(vals ...string) *Param
Values
Restricts an Enum parameter to the listed allowed values. Produces a CUE string disjunction. Combine with .Default() to set a pre-selected value.
Go — defkit
defkit.Enum("policy").
    Values("Retain", "Delete", "Recycle").
    Default("Retain")
CUE — generated
policy: *"Retain" | "Retain" | "Delete" | "Recycle"
.Min(n int) / .Max(n int) *Param
Min / Max
Applies inclusive range constraints to Int parameters. Generates CUE constraint expressions using >= and <= intersected with the base type.
Go — defkit
defkit.Int("replicas").Default(1).Min(1).Max(100)
defkit.Int("timeout").Default(30).Min(5)
CUE — generated
replicas: *1  | int >=1 & <=100
timeout:  *30 | int >=5
.WithFields(params ...*Param) *Param
WithFields
Defines the named fields of an Object or Struct parameter. Each field is itself a fully-configured *Param, enabling nested structures of arbitrary depth.
Go — defkit
probe := defkit.Object("livenessProbe").WithFields(
    defkit.String("path").Default("/healthz"),
    defkit.Int("port").Default(8080),
    defkit.Int("initialDelaySeconds").Default(10),
)
CUE — generated
livenessProbe?: {
    path:                *"/healthz" | string
    port:                *8080 | int
    initialDelaySeconds: *10   | int
}
.Of(paramType *Param) *Param
Of
Sets the element type of an Array. Pass any *Param — scalar, struct, or complex — as the element schema. The element type does not need a name when passed to .Of().
Go — defkit
// Array of strings
tags := defkit.Array("tags").Of(defkit.String(""))

// Array of structs
mounts := defkit.Array("volumeMounts").Of(
    defkit.Struct("").WithFields(
        defkit.String("name").Required(),
        defkit.String("mountPath").Required(),
        defkit.Bool("readOnly").Default(false),
    ),
)
CUE — generated
tags?: [...string]

volumeMounts?: [...{
    name:      string
    mountPath: string
    readOnly:  *false | bool
}]
.Ignore() *Param
Ignore
Marks the parameter as internal — it is excluded from the generated CUE schema and will not appear in vela show or OpenAPI output. Use for deprecated fields or helper variables that exist only in Go for template composition.
Go — defkit
// Deprecated field — excluded from CUE schema
port := defkit.Int("port").
    Ignore().
    Description("Deprecated field, please use ports instead")

return defkit.NewComponent("webservice").
    Params(image, port, ports).  // port excluded from schema
    Template(webserviceTemplate)
CUE — generated
// Only non-ignored params appear:
parameter: {
    image: string
    ports?: [...]
    // port is omitted from schema
}
